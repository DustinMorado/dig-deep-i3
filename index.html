Digging Deep into I3

I'm writing this to try and map out what I3 is really doing, and unwrapping some of the heavier layers of abstraction at least at the level of shell scripts.

Since I3 is closed source and distributed binaries. Once you get down to the compiled binaries there's no clue as to what's going on besides using the usage statements, which may as well be written in heiroglyphics.

Project Root Directory

All projects start relative to a directory, which I will refer to as the project root directory. Since I3 suggests using relative paths, I can refer to it as . or project.

i3param.sh

This file is the heart of i3 programs and defines a huge set of shell environment variables that i3 uses for what appears to be everything. The name of the file is stored in the environment variable I3PARAM, and in the subvolume tutorial it is named ./mraparam.sh.

I've documented the list of environment variable as well as the explanations given in the tutorial file here: i3param.sh.
